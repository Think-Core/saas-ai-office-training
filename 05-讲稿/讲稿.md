### `A-10`｜因地制宜：让 Agent 融入企业办公生态

- **讲述内容**：上一页我们讲了，不同大模型面对的材料和任务各有长处。真正进入企业使用时，还要再看一层：Agent 能不能融入大家已经在使用的办公环境。左边的 ChatGPT 和 Claude Desktop 是两种前沿通用智能体，可以在不同平台之间使用。右边从企业日常的办公入口出发：以钉钉为主要协作平台的团队，可以优先了解 QoderWork；依赖微信、企业微信和腾讯办公生态的团队，可以优先了解 WorkBuddy；以飞书协作为主的团队，可以了解扣子 Coze。这里连接的不只是一个 Logo，而是背后的账号、文档、消息和权限。同一生态通常更容易衔接现有账号、文档和协作流程，但不等于自动打通，实际使用前仍要核对企业版能力、连接器和管理员权限。
- **建议时间**：60—75 秒。
- **录屏时机**：无；本页建立 Agent 工具的分类与选择逻辑。
- **观看重点**：左侧是前沿通用智能体，右侧是从企业现有办公入口出发的优先候选；三条关系不是强制采购建议。
- **可跳过内容**：各产品套餐、价格、模型配置和具体安装方法。
- **互动备用话术**：大家所在团队日常主要使用钉钉、企业微信还是飞书？如果准备引入 Agent，最先需要核对的是账号、文档、消息还是权限？

### `A-09`｜知己知彼：看懂大模型的长处与短板

- **讲述内容**：前面讲了大模型是 Agent 的核心能力，但大模型不是一个可以简单排第一、第二的名单。不同模型面对的材料和任务不同：有的更适合同时参考很多资料，有的能理解图片、表格、PDF 和视频，有的更擅长分析判断，有的能自主调用工具、临时编写脚本或执行特定程序。这里列出六家当前常见的国产模型，是为了帮助大家建立识别框架。当 AI 参与具体工作时，可以从四方面观察差异：能同时参考多少资料；能理解哪些类型的内容；能不能熟练使用工具；速度和成本是否适合当前工作。标签只用于初步筛选。真正重要的工作，仍要拿自己的文档、数据、权限和验收标准做一轮比较，再决定使用哪一个。
- **建议时间**：75—90 秒。
- **录屏时机**：无；本页为概念辨析。
- **观看重点**：六张卡不是排名；标签全都并列，帮助按材料和任务筛选。
- **可跳过内容**：参数数量、评测名称、模型内部结构和价格。
- **互动备用话术**：请想一想，你最近的一项工作最需要 AI 帮你处理的是长资料、图表，还是需要连续完成多个步骤？


### `A-08`｜授人以渔：让 AI 学会一类工作的方法

- **讲述内容**：上一页的 MCP 解决的是“把数据和工具接进来”。但连上工具，不等于知道一项工作该怎样做。Skills 解决的是“把怎么做沉淀下来”：把流程、业务规则、参考资料和交付模板整理成可复用的技能包。左上以一个月度经营简报为例，`SKILL.md` 是工作入口，告诉 AI 何时使用、按什么步骤做；`reference/` 是参考资料目录，放口径、清单与案例；`templates/` 是输出模板目录，固定交付格式；`scripts/` 是程序脚本目录，承担精确的计算、转换与数据处理。右图说明两者互补：Skill 规定方法，MCP 连接文档、数据和业务系统。左下的典型 Agent 工作流不是让 AI 自动越权，而是先按 Skill 读取规则与模板，再在授权范围内通过 MCP 取用所需资料，最后交回可复核的初稿。可以用页面结论收束：Skills 把完成特定任务所需的领域知识、操作流程、要用的工具及最佳实践都封装在一起。右下展示的是发现技能的入口和办公场景示例；安装前仍要确认发布者、依赖权限、数据去向和适配版本。
- **建议时间**：75—90 秒。
- **录屏时机**：无；本页只完成概念解释，为后续 WorkBuddy 技能入口和案例做铺垫。
- **观看重点**：四个文件夹不是四种产品，而是一套工作方法的不同组成部分；Skills 与 MCP 是“方法 + 连接”的关系。
- **可跳过内容**：不解释 YAML、脚本语言、Agent Host、协议与目录安装方式。
- **互动备用话术**：请想一项你每月都要反复交代给 AI 的工作：它的流程、口径和模板，是否已经可以被整理成一个 Skill？

### `A-03`｜完成一项工作，背后要做多少事情？

- **讲述内容**：完成一项工作，从想清楚到交出去，中间会有很多环节需要反复沟通和处理。先把事情说清楚，再找资料、找素材，和人讨论、补信息、改内容；有时改到一半，又要回头确认前面的要求。做完之后，如果没有把有用的材料和结论留下来，下次还是得重新走一遍流程。这些事不难，但很占精力，也不只发生在汇报里。下一页我们再看，怎样把这些环节组织得更顺。
- **建议时间**：45—60 秒。
- **录屏时机**：无。
- **观看重点**：四个卡点分别对应任务起步、信息获取、协作修改和经验复用；中央成果页强调工作会在不同环节间来回推进，本页只建立共鸣，不讲解决方案。
- **可跳过内容**：无。
- **互动备用话术**：请大家想一想，最近一项工作里，自己最常遇到的是“需求说不清楚、信息搜集不全、总是反复调整、成果无法复用”中的哪一种？

### `A-04`｜AI 能帮我们做什么？

- **讲述内容**：上一页我们看到，一件工作不只是写出一个结果，里面还有找资料、理材料、沟通和反复确认。AI 的能力也不是一下子就到了今天。2022 到 2024 年，我们更多是在问一个问题、拿一个回答；到 2025 年，可以把一批材料交给它，让它先理出重点、起一版初稿，但它还不能自主决策、创造工具、循环执行，后续仍需要人推动。现在，我们开始可以把“要完成什么、可参考什么、怎样算完成”交代清楚，让 AI 围绕这些要求参与推进其中多个环节。它不是顺着一条固定流程走：会先判断还缺什么，再去找、整理、生成或修改；需要时使用或创造工具，边做边核对，把过程和可检查的结果交回来。最后的判断仍然在人手上：目标怎么定、边界在哪里、有哪些风险、结果能不能用，都由人决定。当 AI 不只是回答问题，而是能围绕目标和要求推进任务时，我们开始把它称为 Agent，也就是智能体。
- **建议时间**：50—70 秒。
- **录屏时机**：无。
- **观看重点**：三段是 AI 能力在不同时间的变化，不是一件任务的三个步骤；右上任务单说明 Agent 的输入是目标、材料、要求和边界，人的职责是检查并决定是否采用。
- **可跳过内容**：第一、二阶段的示意细节；保留第三阶段与结论即可。
- **互动备用话术**：大家可以想想，自己平时用 AI 最多是在“问一个问题”，还是已经会把一批材料和一个明确目标交给它？

### `A-05`｜AI 工具 / 助手发展简史

- **讲述内容**：刚才我们看到，AI 今天已经可以参与推进任务。这种工作方式是一步步发展出来的。2022 年，ChatGPT 让更多人第一次大规模体验到自然对话、改写和内容生成；2023 到 2024 年，Copilot、Notion AI 一类助手进入文档、表格、知识整理和代码等具体工具，AI 开始帮助完成其中一段工作，也开始出现 Tool Calling（调用工具）这类能力。2025 年，RAG、MCP、Skill、Computer Use、Browser Use 等能力受到关注，资料、网页和工具可以更容易进入同一次处理。到了现在，Agent 开始围绕人交代的目标、材料、权限和验收要求处理多个环节，并把过程与结果交回给人检查。目标怎么定、边界在哪里、有哪些风险、结果能不能用，仍由人负责。
- **建议时间**：60—75 秒。
- **录屏时机**：无。
- **观看重点**：四张卡表达的是产品形态与能力条件的演进，不是同一项任务的四个步骤；`RAG` 放在 2023—2024 的企业资料问答与知识连接语境，`MCP` 作为 2025 年工具连接讨论的重要技术标签。
- **可跳过内容**：前两张卡的举例细节；保留 2025、现在与页面结论即可。
- **互动备用话术**：大家现在用 AI 时，手头的资料、网页和工具，是否还散在对话框之外？

### `A-06`｜大模型和 Agent，分别在做什么？

- **讲述内容**：前面我们已经看到，AI 可以从回答问题走向参与推进任务。这里把两个概念捋清。大模型负责理解信息、生成内容、调用工具并给出建议。Agent 是一个以大模型为核心的软件，围绕目标组织任务。作为一种工作模式，Agent 给大模型提供材料、工具，并响应大模型的反馈。最后仍由人机协同完成：人确定目标与边界、判断风险、验收结果。后面再遇到知识库、Skill 或工具调用这些词，就可以把它们放回这张任务框架里理解。
- **建议时间**：55—65 秒。
- **录屏时机**：无。
- **观看重点**：大模型是 Agent 的核心能力，Agent 是围绕任务组织这份能力的工作框架；两者不是并列替代关系。
- **可跳过内容**：左侧“一次输入—生成回答”的视觉细节；保留中央核心、四项任务条件和人的职责即可。
- **互动备用话术**：大家平时把 AI 当作“问一句、答一句”的工具多一些，还是已经会把目标、资料和完成标准一起交代给它？

### `A-07`｜工欲善其事，必先利其器。

- **讲述内容**：上一页讲 Agent 怎样围绕目标组织任务。要把任务真正往前推进，还得有合适的“器”。MCP 的全称是 Model Context Protocol，中文叫模型上下文协议；它不是一个具体工具，而是一套让 AI 连接数据与工具的通用协议。过去，要逐个适配工具；MCP 就像拓展坞，让双方遵循同一套连接方式。它让 AI 在授权范围内读取信息、调用工具，再把结果带回任务。右图只用来说明角色关系：AI 应用经 MCP 客户端连接服务器，再由服务器接出文档、消息、知识库、数据库等服务；图中的 WorkBuddy 是客户端角色示意，不代表现场产品配置。企业里常见两种形态：本地 MCP 访问已授权的本地内容；网络 MCP 放在内网、私有云或服务平台，供团队共享。右下角给出两个发现入口和四类办公场景示例，最下方提醒大家：MCP 服务接入前先核对来源、权限范围和数据去向。
- **建议时间**：75—90 秒。
- **录屏时机**：无。
- **观看重点**：左图重点是“统一接口像拓展坞”；右图重点是“客户端—服务器—外部服务”；再区分本地与网络两种连接形态。
- **可跳过内容**：JSON-RPC、stdio、Streamable HTTP 的实现细节；只保留它们分别对应本地与网络连接的直觉。
- **互动备用话术**：如果让你常用的 AI 帮你完成一项工作，你最希望它连接哪一类信息或工具：文件、网页、业务系统，还是知识库？
