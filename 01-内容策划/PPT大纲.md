# PPT 大纲与页数预算

## 页数原则

- 当前只确定章节、讲述时间和页面预算，不提前填具体页面内容。
- 普通认知页按约 1—2 分钟一页估算；一页只承担一个清楚的讲述任务。
- 每个案例先用 3—4 页把场景讲清楚，再用 1 页独立播放录屏；需要时可在视频后增加 1 页结果复盘。
- 案例具体材料、流程和结论必须在实际研究与录屏准备过程中共同确定，不凭空编造。
- 当前基准方案约 42—44 页，按材料复杂度允许在约 38—49 页之间调整；这是页面预算，不是最终必须达到的数字。

## 第一章｜开场与课程任务

- 建议时间：约 3 分钟。
- 页面预算：5 页。
- 作用：完成讲师定位、课程问题和学习目标。
- `A-01` 封面已确定：显示标题“AI Agent 赋能基础办公”，右侧使用 Codex、Claude Desktop、Coze、WorkBuddy、Manus、QoderWork 高清官方图标矩阵；不显示副标题、底部总结句、学习路径和媒体标签，右上角使用人民邮电出版社科技蓝 Logo。
- `A-02` 学习路线已确定：不显示“课程目录”等页名，使用四行纵向结构呈现“理解 Agent—上手智能体工具—掌握人机协作—办公场景实战”；右侧以四层上升的抽象科技蓝几何图形平衡版面，不增加文字。
- `A-03` 课程问题已确定：以“完成一项工作，背后要做多少事情？”为题，用“需求说不清楚、信息搜集不全、总是反复调整、成果无法复用”四个日常卡点建立共鸣；不点名具体场景、不出现 Agent，下一页再进入概念解释。
- `A-04` 能力演进已确定：以“AI 能帮我们做什么？”为题，按 2022—2024、2025、现在三个时间阶段说明 AI 从回答问题、整理材料到参与推进任务的变化；2025 阶段可读一批资料、梳理重点和起草内容，但还不能自主决策、创造工具或循环执行；右上任务单以“判断还缺什么—找、整理、生成或修改—边做边核对并交回结果”说明 Agent 的办公推进逻辑，其中需要时可使用或创造工具，过程和结果都应可检查；人负责选目标、定边界、看风险和最终验收。页面末尾引出“Agent（智能体）”，下一页解释这种变化出现的背景。
- `A-05` 发展简史已确定：以“AI 工具 / 助手发展简史”为题，采用 2022、2023—2024、2025、2026 至今四阶段，说明 AI 如何从对话与生成，进入常用工具，连接资料与工具，进而让 Agent 参与任务推进。页面保留 ChatGPT、Copilot、RAG、Tool Calling、MCP、Skill、Computer Use、Browser Use 等少量技术锚点，但全部解释改为办公语言；下一页再正式解释大模型与 Agent 的差异。
- `A-06` 概念辨析已确定：以“大模型和 Agent，分别在做什么？”为题，用嵌套关系而非左右对照说明两者关系。左侧四句结论分别说明：大模型理解信息、生成内容、调用工具并给出建议；Agent 是以大模型为核心、围绕目标组织任务的软件；Agent 作为工作模式向大模型提供材料和工具并响应反馈；人机协同确定目标与边界、判断风险、验收结果。本页不再复述时间演进或引入 MCP、RAG 等术语。
- 开场五页已承担课程定位、学习路线、课程问题、能力演进与发展背景，不再为满足页数增加空页。

## 第二章｜从大模型到 Agent

- 建议时间：约 8 分钟。
- 页面预算：5—6 页。
- 作用：用足够页面分别解释演进、产品形态、Agent 构成和工具版图。
- 每个概念尽量独立成页，避免把时间轴、产品对比、架构图和结论挤在同一页。

## 第三章｜WorkBuddy 从安装到可用

- 建议时间：约 10—12 分钟。
- 页面预算：7—9 页。
- 作用：在任何案例之前讲清定位、安装、首次配置、权限、界面、工作目录、模式、Skills 和 Connector。
- 页面数量根据实际客户端界面和可用截图决定，不为凑页数拆分。

## 第四章｜看懂案例的统一方法

- 建议时间：约 4 分钟。
- 页面预算：2—3 页。
- 作用：说明如何定义成果、准备上下文、选择模式、指定输出、观察执行和验收结果。

## 第五章｜四个核心案例

- 建议时间：合计约 28—32 分钟。
- 页面预算：每个案例 5 页为基础，必要时增加至 6 页；合计约 20—24 页。

每个案例使用以下结构槽位：

1. **场景页**：为什么要做这件事，原流程哪里费时或容易出错。
2. **输入与成果页**：有哪些材料，最终要交付什么，给谁使用。
3. **方法或流程页**：WorkBuddy 如何参与，人和 AI 如何分工。
4. **验收与观看提示页**：怎样判断结果可用，录屏时重点看什么。
5. **独立录屏页**：以视频为主体，不与大段说明挤在一页。
6. **可选复盘页**：只有案例结果复杂、需要前后对照或总结方法时才增加。

四个案例方向保持不变：

- 从零散工作记录生成周报；
- 多份材料生成阶段性工作总结；
- 营销策划与配套素材；
- 从零散资料形成可持续使用的知识库。

现阶段只保留这些结构槽位。做到某个案例时，再共同寻找材料、确认真实场景、设计录屏并填写页面内容。

## 第六章｜低优先级现场加演

- 建议时间：0—4 分钟。
- 页面预算：1—2 页。
- 默认可整段跳过。
- 具体采用安全默认题还是现场听众选题，在录屏和案例主线完成后再决定。

## 第七章｜收尾

- 建议时间：约 2 分钟。
- 页面预算：1—2 页。
- 回扣课程方法和学员下一步行动，不使用只有“谢谢”的空结束页。
