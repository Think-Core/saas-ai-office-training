## 制作与协作想法

- 不把 PPTX 当作唯一源文件，优先制作网页版演示并进行 Git 管理。
- HTML 可使用 ECharts、动画、视频和 GIF；之后再导出或转换为 PPTX。
- PPTX 是阶段性交付成果，过程设计和内容以文本、JSON、HTML 和可编辑图示管理。
- 多个 task 会讨论不同章节，所有 task 必须通过文档获得最新进展。
- 原始想法、大纲、素材、讲稿和阶段成果分别管理。
- 网页课件是代码，只维护一份当前源代码和一份当前讲稿，通过同一个 Git commit/tag 保证对应，不为每个版本复制目录。
