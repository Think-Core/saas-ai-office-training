# Codex 与办公案例资料

## 在本课程中的角色

Codex 不是四个核心案例的主工具。它主要承担：

- Agent 工具版图中的能力对照；
- 展示长期项目上下文、跨 task 协作、版本管理和可验证交付；
- 在 WorkBuddy 不适合的复杂文件或项目型任务中作为备选案例。

## 可用的项目级案例

当前课件项目本身就是一个可追溯示例：

- `AGENTS.md` 统一不同 task 的工作规则；
- 项目进展、原始想法、大纲和页面台账共同维护上下文；
- HTML、讲稿、PPTX、PDF 使用统一版本号；
- Git 保存过程变化，Git LFS 管理二进制成果；
- 每个 task 更新交接记录，减少依赖聊天记忆。

## 是否进入主课件的判断条件

只有同时满足以下条件才进入：

1. 能在 2—3 分钟内说清；
2. 与业务骨干的工作方式有关，而非只展示编程；
3. 能体现 WorkBuddy 主案例未覆盖的能力；
4. 不挤占 WorkBuddy 安装教学和四个核心案例的时间。
