# WorkBuddy Skills 与 Connector 资料

本文件服务于课件第 10 页和案例录屏准备。

## 教学定义

- Skill 回答“应该怎样完成这类任务”，承载专业方法、步骤、模板和输出标准。
- Connector 回答“可以连接并调用什么”，让 Agent 在授权范围内访问外部服务或工具。
- Skill 不应简单等同于一条长提示词；它的价值在于可复用的方法和配套材料。

以上概念可由 WorkBuddy 产品生态白皮书辅助理解，但公开课件应优先采用官方公开文档和实机界面。

## 第 10 页建议结构

左侧展示 Skill：

- 技能市场入口；
- 搜索、安装、启用或卸载；
- 用户级与项目级影响范围，如当前版本存在该区分；
- 不要一次安装大量无关技能。

右侧展示 Connector/插件：

- 连接外部服务的目的；
- 授权范围和凭证安全；
- 调用前后的确认机制；
- 不在课堂上配置包含真实企业数据的连接器。

## 本课程最小能力组合

这里先定义能力类别，不提前写死产品名称：

1. 读取和生成 Word、PDF、Markdown 等文档；
2. 读取表格和结构化数据；
3. 文案策划、改写或多渠道内容生成；
4. 文件整理、知识提取或知识库能力；
5. 必要的本地文件访问与输出保存权限。

## 常见问题

- 技能装得太多，Agent 可能选错能力或让界面变复杂。
- 已有 Skill 时仍塞入过长提示词，可能造成方法冲突。
- Skill 或 Connector 更新后，旧截图和旧录屏可能失效。
- Connector 授权范围过大，会扩大误操作和数据暴露风险。
