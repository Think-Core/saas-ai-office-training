# 课件视觉验收与浏览器操作约定

适用范围：所有涉及网页课件预览、版式调整、可编辑文字、缩略图、页面跳转、导出前检查的 task。

## 唯一验收原则

网页课件是视觉与文字共同构成的交付物。不能只根据 HTML 或 CSS 推断效果；每次涉及页面内容或版式的修改，都必须在实际课件页面中检查：

- 字号与信息层级是否适合投屏；
- 留白、对齐、遮挡、裁切和图标位置是否正常；
- 左侧目录、缩略图、翻页计数与页面顺序是否一致；
- 用户在网页编辑器中修改过的文字是否仍可直接编辑，且已同步回源文件。

闭环顺序：`修改 → 当前 Chrome 实际预览 → 画面复核 → 同步课件源文件与项目记录`。

## 浏览器操作约定

- 用户的 Chrome 已安装并启用可用的控制扩展；涉及课件预览、验收或编辑器状态时，优先直接连接**用户当前已打开的课件标签页**。
- 不因“未开启调试端口”而要求用户重启 Chrome、重新打开课件，或改用另一套浏览器窗口。调试端口只是备用方式，不是使用当前 Chrome 的前置条件。
- 继续使用用户正在看的本地预览地址：`http://127.0.0.1:5311/`。除非用户明确要求，不新开浏览器窗口、不切换浏览器、不替换用户当前标签页。
- 操作前先确认当前标签页和页面状态；完成后保留该标签页，停留在本次验收的页面，方便用户继续查看或编辑。
- 浏览器控制仅用于必要的页面查看、刷新、导航和验收，不读取账号、Cookie、密码或个人浏览数据。

## 源文件同步

- 用户在网页编辑器中改过的内容，以当前页面的保存状态为准；随后同步 `04-网页版PPT/课件源代码/index.html`、`goal.json`、讲稿和关联项目记录。
- 新增或调整页面后，必须同时检查目录页数、缩略图、页码与实际页面数一致，避免“页面已存在但目录未统计”的情况。
